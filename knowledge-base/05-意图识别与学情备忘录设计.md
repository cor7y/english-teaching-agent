# 意图识别分类体系与学情备忘录设计

本文档定义了教学Agent的意图识别分类体系、学情备忘录数据结构和闭环运作逻辑。

设计依据：分类体系来自00-taxonomy-and-mvp.md；mastery计算参考贝叶斯知识追踪（BKT）模型；能力层级参考布鲁姆认知目标分类（Bloom's Taxonomy）

---

## 一、意图识别分类体系

### 第一层：问题性质（Intent Type）

| 代码 | 类型 | 描述 | 典型输入 |
|------|------|------|---------|
| K | 知识查询 | 学生想了解某个知识点的规则/定义/用法 | "定语从句怎么用""什么时候用现在完成时" |
| D | 错误诊断 | 学生提交了一个句子/答案，想知道哪里错了、为什么错 | "这个句子哪里错了""为什么这里不能用that" |
| S | 解题求助 | 学生遇到一道具体的题目，需要解题指导 | "这道语法填空怎么做""帮我分析一下这道完形" |
| M | 方法策略 | 学生想了解学习方法、解题技巧、复习规划 | "怎么背单词""语法填空有什么技巧""怎么安排复习" |
| P | 练习请求 | 学生想做练习题或自测 | "给我出几道题""我想练习非谓语动词" |
| E | 情感表达 | 学生表达学习中的情绪（焦虑、沮丧、困惑） | "英语太难了""我怎么总是做错" |

### 第二层：知识维度（Knowledge Domain）

1. 词汇 (Vocabulary)
   1.1 词义理解（含熟词生义）
   1.2 词汇辨析（近义词/形近词）
   1.3 词性与构词法（词根词缀）
   1.4 固定搭配与短语
   1.5 拼写

2. 语法 (Grammar)
   2.1 时态与语态
   2.2 非谓语动词（不定式/动名词/分词）
   2.3 从句（定语从句/名词性从句/状语从句）
   2.4 特殊句式（倒装/强调/虚拟语气/省略）
   2.5 主谓一致
   2.6 冠词与介词
   2.7 代词
   2.8 情态动词

3. 语篇 (Discourse)
   3.1 篇章结构
   3.2 衔接手段
   3.3 文体特征
   3.4 主旨与段意概括

4. 语用 (Pragmatics)
   4.1 语境理解
   4.2 语言得体性
   4.3 中式英语/母语负迁移

5. 题型技能 (Test Skills)
   5.1 阅读理解策略
   5.2 七选五策略
   5.3 完形填空策略
   5.4 语法填空策略
   5.5 读后续写策略
   5.6 应用文写作策略
   5.7 听力策略

6. 学习策略 (Learning Strategy)
   6.1 记忆方法
   6.2 复习规划
   6.3 时间管理
   6.4 自我监控

### 第三层：能力层级（Proficiency Level）

| 层级 | 描述 | 对应认知目标 |
|------|------|-------------|
| L1 识记 | 知道规则/定义 | 记忆、识别 |
| L2 理解 | 能解释和区分 | 理解、比较 |
| L3 应用 | 能在新语境中使用 | 运用、实践 |
| L4 分析 | 能识别错误和原因 | 分析、评价 |

## 二、意图识别逻辑

### 识别流程
```
用户输入
  ↓
Step 1: 识别问题性质（K/D/S/M/P/E）
  - 关键词匹配 + 句式模式识别
  - "怎么用/什么是/区别" → K
  - "哪里错了/为什么错/对不对" → D
  - "怎么做/帮我分析/这道题" → S
  - "怎么背/技巧/方法/复习" → M
  - "出题/练习/测试" → P
  - "太难了/学不会/好烦" → E
  ↓
Step 2: 识别知识维度（1.x-6.x）
  - 从用户输入中提取知识点关键词
  - 映射到分类体系的具体节点
  ↓
Step 3: 判断能力层级（L1-L4）
  - 根据问题的深度和复杂度判断
  ↓
Step 4: 生成结构化标签
  - 格式：{intent: "K", domain: "2.2", level: "L2"}
  ↓
Step 5: 录入学情备忘录 + 生成响应
```

### 意图识别示例

| 用户输入 | intent | domain | level | 响应策略 |
|---------|--------|--------|-------|---------|
| "现在完成时怎么用" | K | 2.1 | L1 | 讲解规则+例句 |
| "have been和had been有什么区别" | K | 2.1 | L2 | 对比讲解+语境示例 |
| "这个句子为什么用doing不用to do" | D | 2.2 | L4 | 分析具体语境+规则解释 |
| "帮我看看这道语法填空" | S | 待分析 | L3 | 引导解题思路 |
| "语法填空怎么提分" | M | 5.4 | - | 提供解题策略 |
| "给我出5道非谓语动词的题" | P | 2.2 | - | 生成练习题 |
| "英语语法好难啊" | E | 2.x | - | 情感支持+引导到具体问题 |

### 更新规则

#### mastery值计算
- 初始值：0.0（unknown状态）
- 答对一题：+0.1（上限1.0）
- 答错一题：-0.15（下限0.0）
- 连续3次答对：mastery直接设为max(当前值, 0.8)，status→"mastered"
- 连续2次答错：status→"weak"，触发专项讲解
- 7天未练习：mastery自然衰减-0.05/周

#### status状态机
```
unknown → (首次答题) → learning
learning → (连续3次答对) → mastered
learning → (连续2次答错) → weak
mastered → (7天未练习) → learning
weak → (答对) → learning
```

#### error_history记录格式
```json
{
  "date": "2026-04-24",
  "error_type": "时态混用：一般过去时误用为现在完成时",
  "context": "语法填空第3题：He _____(live) here since 2020. 学生填lived，正确答案has lived",
  "domain": "2.1",
  "sub_point": "present_perfect_vs_past"
}
```

## 四、闭环运作逻辑

### 完整闭环流程

```
1. 用户输入
   ↓
2. 意图识别 → 生成标签 {intent, domain, level}
   ↓
3. 查询学情备忘录 → 获取该domain的当前mastery和error_history
   ↓
4. 生成响应（根据intent类型）
   - K: 讲解知识点（根据level调整深度）
   - D: 分析错误原因 → 更新error_history
   - S: 引导解题 → 根据结果更新mastery
   - M: 基于学情推荐策略
   - P: 生成针对weak点的练习题
   - E: 情感支持 → 引导到具体问题
   ↓
5. 更新学情备忘录
   ↓
6. 生成反馈
   - 即时反馈："这道题考的是非谓语动词，你在这个知识点上已经连续2次出错了"
   - 周期反馈："本周你的语法填空正确率从60%提升到72%"
   ↓
7. 推荐下一步
   - 基于mastery排序推荐最薄弱的知识点
   - 或基于error_history推荐相似题型
```

### MVP验证指标

1. 意图识别准确率：目标>85%（在K/D/S/M/P/E六类中正确分类）
2. 知识维度定位准确率：目标>80%（正确识别涉及的语法知识点）
3. 学情备忘录更新一致性：每次交互后备忘录状态正确更新
4. 闭环完整性：从输入→识别→记录→反馈→推荐的完整链路可跑通
